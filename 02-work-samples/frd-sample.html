1. Introduction

1.1 Purpose

This Functional Requirements Document (FRD) defines the detailed functional specifications for the Remote Patient Monitoring (RPM) Platform. It describes the system behaviors, use cases, business rules, and data requirements that the development team must implement.

1.2 Scope

This FRD covers the Patient Vitals Recording module, including device integration, data validation, alert generation, and clinical review workflows. It complements the Business Requirements Document (BRD) and Product Requirements Document (PRD) for this project.

2. Functional Requirements

2.1 Patient Vitals Recording (FR-VIT)

Req ID Requirement Acceptance Criteria Priority
FR-VIT-001 System shall receive vital readings from BLE-connected medical devices (BP monitor, pulse oximeter, glucometer) Device data received within 5 seconds of transmission; all readings stored with timestamp High
FR-VIT-002 System shall validate vital readings against clinically acceptable ranges before storage Out-of-range values flagged; validation rules configurable per vital type High
FR-VIT-003 System shall allow manual entry of vital readings with mandatory timestamp Manual entries marked with "Manual" source flag; requires patient confirmation Medium
FR-VIT-004 System shall display vital trends in graphical format (7-day, 30-day, 90-day views) Charts render within 2 seconds; support zoom and pan interactions Medium
FR-VIT-005 System shall generate alerts when vitals exceed patient-specific thresholds Alerts generated within 30 seconds; notification sent to assigned clinician High

2.2 Clinical Review Workflow (FR-REV)

Req ID Requirement Acceptance Criteria Priority
FR-REV-001 System shall present daily patient queue sorted by alert priority (Critical > High > Medium > Low) Queue refreshes every 60 seconds; supports manual refresh High
FR-REV-002 System shall display patient summary card with latest vitals, trends, and alert history Summary loads within 3 seconds; shows last 7 days of readings High
FR-REV-003 System shall allow clinician to document review notes with structured templates Templates include: Assessment, Plan, Follow-up; auto-save every 30 seconds Medium
FR-REV-004 System shall track time spent on each patient review for CPT billing Timer starts on patient card open; pauses on inactivity > 2 minutes High
FR-REV-005 System shall require clinician sign-off to complete review Digital signature captured; timestamp recorded; review locked after sign-off High

3. Use Cases

Record Patient Vitals via BLE Device

UC-VIT-001
Actor
Patient
Preconditions
Patient has registered BLE device paired with mobile app; Patient is enrolled in RPM program
Main Flow
  1. Patient opens RPM mobile app and navigates to "Record Vitals"
  2. System displays list of paired devices with connection status
  3. Patient selects device (e.g., Blood Pressure Monitor) and taps "Start Reading"
  4. System initiates BLE connection and displays "Connecting..." status
  5. Patient takes measurement using the physical device
  6. Device transmits reading to app via BLE
  7. System validates reading against acceptable ranges
  8. System displays reading with visual indicator (Normal/Elevated/Critical)
  9. System stores reading and syncs to cloud server
  10. System confirms "Reading Saved Successfully"
Alternative Flow (A1 - Out of Range)
At step 7, if reading is outside acceptable range, system displays warning and prompts patient to retake measurement or confirm reading is accurate
Alternative Flow (A2 - Connection Failed)
At step 4, if BLE connection fails after 30 seconds, system displays error and offers manual entry option
Postconditions
Vital reading stored in patient record; Alert generated if thresholds exceeded; Reading available for clinical review

Complete Clinical Review

UC-REV-001
Actor
Clinical Staff (RN/MA)
Preconditions
Clinician is logged into provider portal; Patient has pending vitals requiring review
Main Flow
  1. Clinician views daily patient queue on dashboard
  2. System displays patients sorted by alert priority with visual indicators
  3. Clinician selects patient from queue
  4. System opens patient summary card and starts review timer
  5. Clinician reviews vital trends, alerts, and historical data
  6. Clinician documents assessment using structured template
  7. Clinician selects follow-up action (No Action / Schedule Call / Escalate to Provider)
  8. Clinician clicks "Complete Review" and provides digital signature
  9. System stops timer and records total review time
  10. System moves patient to "Reviewed" status and updates queue
Postconditions
Review documented with timestamp; Review time logged for billing; Patient removed from pending queue

4. Business Rules

BR-001: Vital readings older than 24 hours from device timestamp shall be flagged as "Delayed" and excluded from real-time alert processing.
BR-002: A patient can have only ONE active device per vital type (e.g., one BP monitor, one glucometer). Registering a new device of same type deactivates the previous device.
BR-003: Clinical reviews must be completed within 24 hours of alert generation for Critical alerts, 48 hours for High alerts, and 72 hours for Medium/Low alerts.
BR-004: CPT code 99458 (RPM - each additional 20 minutes) can only be billed if cumulative monthly review time exceeds 20 minutes for a patient.
BR-005: Patient consent for RPM enrollment must be documented and stored before any vital data collection begins. Consent expires after 12 months and requires renewal.
BR-006: All vital readings must be retained for minimum 7 years per HIPAA retention requirements.

6. Interface Requirements

Interface Type Description
BLE Device SDK Hardware API Integration with mio-labs SDK for receiving vital readings from certified medical devices. Supports BP monitors, pulse oximeters, glucometers, and weight scales.

7. Appendix

7.1 Vital Range Thresholds (Default)

Vital Type Normal Elevated High Critical
Blood Pressure (Systolic) < 120 mmHg 120-139 mmHg 140-179 mmHg ≥ 180 mmHg
Heart Rate 60-100 bpm 50-59 or 101-110 bpm 40-49 or 111-130 bpm < 40 or > 130 bpm
SpO2 ≥ 95% 92-94% 88-91% < 88%
Blood Glucose (Fasting) 70-100 mg/dL 101-125 mg/dL 126-200 mg/dL < 70 or > 200 mg/dL
Note: These are default thresholds. Clinicians can configure patient-specific thresholds based on individual health conditions and treatment plans.
