2.1 Device Management Module Healthcare RPM

FeatureSub-Features / Business RulesPriority
Device Inventory
  • Device registration with ICCID, IMSI, IMEI tracking
  • Battery level monitoring (100%, 75%, 50%, 25%, Low)
  • Signal strength indicator (25%, 50%, 75%, 100%)
  • Device status: Active | Shipped | Out of Service | Returned
  • Vendor integration with mio-labs.com for device data
 High
Device Allocation
  • Allocate from PDC inventory → Provider → Patient
  • Link device to patient record with allocation date
  • Track shipping status and carrier information
  • Device can only be allocated to one patient at a time
  • Deallocation requires admin approval if active readings exist
 High
External Care Providers
  • Directory of Specialists, PCPs, Dieticians by NPI number
  • Share access: Grant temporary read access to patient data
  • Revoke access: Remove shared access with audit log
  • View profile: See provider credentials and contact info
  • Specialty types: Dentist, Nutritionist, Dietician, Cardiologist
 Medium
Bulk Patient Onboarding
  • Download CSV template with required fields
  • Fields: Name, DOB, Phone, Email, POP ID, Insurance
  • Validation: Check for duplicates, required fields, format
  • Error report: Row-by-row validation errors
  • Success: Create patient records, assign to POP
 Medium
Reading Sync
  • Auto-sync readings within 5 seconds of device transmission
  • Display last reading timestamp: "Today, 09:03 PM EST"
  • Reading types: BP (mmHg), SpO2 (%), Glucose (mg/dL), Weight (lbs)
  • Out-of-range alert triggers within 30 seconds
 High

2.2 Appointment Booking Scenarios Telehealth UK

ScenarioSteps / Rules
1. Patient Self-Booking 1. Login to portal/app → 2. Browse providers by specialty → 3. View available slots → 4. Select type: New Patient (45 min) or Follow-up (15 min) → 5. Complete intake form → 6. Receive confirmation + video link → 7. Reminders: 24h + 1h before
2. Admin Books for Patient 1. Search patient by Name/DOB/Phone → 2. If new: Create record first → 3. Select provider, view calendar → 4. Book slot, add notes (e.g., "needs translator") → 5. System sends confirmation to patient
3. Provider Follow-up 1. Provider marks "Follow-up in X days" → 2. System suggests available slots → 3. Provider selects OR sends 3 options to patient → 4. Patient picks preferred time via link
4. Recurring Series: Same slot weekly for X sessions (therapy)
Auto: System suggests next per care plan (e.g., 3 months)
5. Rescheduling By Patient: Via portal if >24 hrs before
By Clinic: Select reason, notify patient + offer alternatives
By Provider: Mark day unavailable, batch reschedule
6. Cancellation >24 hrs: No fee, slot released to waitlist
<24 hrs: Late fee may apply (configurable)
No-show: Fee charged, flagged in patient record
7. Waitlist 1. Patient joins waitlist with preferred times → 2. Slot opens (cancellation) → 3. System notifies waitlist in order → 4. First to confirm (within 2 hrs) gets slot
8. Day-of Check-in 1. Patient receives check-in link (15 min before) → 2. Update intake, confirm insurance, sign consent → 3. Enter virtual waiting room → 4. Provider admits to video call → 5. After call: Status "Completed", billing triggered
