1. Introduction

This document showcases user stories from multiple real-world projects across different domains. Each story follows industry best practices with clear acceptance criteria using the Gherkin (Given-When-Then) format.

Story Format Used

Standard Format: "As a [role], I want [feature], so that [benefit]"

Acceptance Criteria: Given-When-Then (Gherkin) format for testability

  • Given - The precondition or context
  • When - The action or trigger
  • Then - The expected outcome

Healthcare

Remote Patient Monitoring Platform

RPM-101
Feature High 8 pts
As a patient with a chronic condition,
I want to record my daily vital signs through my mobile device,
So that my healthcare provider can monitor my health remotely and detect issues early.
Acceptance Criteria
Given the patient is logged into the mobile app
When they navigate to "Record Vitals"
Then they see input fields for blood pressure (systolic/diastolic), heart rate, blood glucose, weight, and oxygen saturation
Given the patient enters vital readings
When they submit the form
Then readings are validated against acceptable ranges and stored with timestamp
Given a reading falls outside normal range
When the submission completes
Then an alert is generated for the assigned care team
Note: HIPAA compliance required - all data must be encrypted at rest and in transit. Audit logging enabled for all vital submissions.
RPM-102
Feature High 13 pts
As a care coordinator,
I want to view a dashboard of all my assigned patients with their latest vital trends,
So that I can quickly identify patients who need immediate attention.
Acceptance Criteria
Given the care coordinator is logged in
When they access the patient dashboard
Then they see a list of all assigned patients with status indicators (normal/warning/critical)
Given a patient has readings in critical range
When dashboard loads
Then that patient appears at the top with red indicator and time since last critical reading
Given dashboard is displaying
When coordinator clicks on a patient row
Then detailed vital history with 7-day trend graphs is displayed
RPM-103
Feature Medium 5 pts
As a physician,
I want to set custom threshold ranges for individual patients,
So that alerts are triggered based on each patient's specific health condition and history.
Acceptance Criteria
Given a physician has a patient assigned
When they access patient settings
Then they can modify alert thresholds for each vital type
Given custom thresholds are saved
When patient submits new vitals
Then system evaluates against custom thresholds, not default values
Given thresholds are modified
When saved successfully
Then change is logged with physician ID, timestamp, and previous values
RPM-104
Task Medium 3 pts
As a patient,
I want to receive medication reminders at scheduled times,
So that I maintain adherence to my prescribed treatment plan.
Acceptance Criteria
Given a medication schedule is configured
When reminder time arrives
Then push notification is sent with medication name and dosage
Given patient receives reminder
When they tap "Mark as Taken"
Then adherence record is logged with timestamp
Given patient misses a dose (no action within 30 min)
When follow-up time elapses
Then missed dose is recorded and care team notified if pattern detected
Fintech

Payroll Management System

PAY-201
Epic High 21 pts
As a payroll administrator,
I want to run end-to-end payroll processing for all employees,
So that salaries are calculated accurately with all deductions and paid on time.
Acceptance Criteria
Given it's the payroll processing date
When admin initiates payroll run
Then system calculates gross pay, deductions (tax, insurance, retirement), and net pay for all active employees
Given payroll is calculated
When admin reviews and approves
Then payment file is generated in bank-compatible format
Given payroll is processed
When processing completes
Then pay stubs are generated and accessible to employees via self-service portal
PAY-202
Feature High 8 pts
As an employee,
I want to view my salary breakdown and download pay slips,
So that I can track my earnings and have records for tax filing.
Acceptance Criteria
Given employee is logged into self-service portal
When they navigate to "My Pay"
Then they see current month salary breakdown: basic, allowances, deductions, net pay
Given employee is viewing pay history
When they click "Download" on any month
Then PDF pay slip is downloaded with company letterhead
Given employee accesses portal
When they select date range
Then they can download consolidated salary statement for specified period
PAY-203
Feature Medium 5 pts
As an HR manager,
I want to configure salary structures with multiple components,
So that different employee grades have appropriate pay structures.
Acceptance Criteria
Given HR manager has admin access
When they create a new salary structure
Then they can define components: basic, HRA, special allowance, conveyance, medical
Given salary structure is created
When they assign it to employee grade
Then all employees in that grade inherit the structure
Given existing structure is modified
When changes are saved
Then system prompts for effective date and applies to future payroll only
PAY-204
Task Low 3 pts
As a finance controller,
I want to generate statutory compliance reports,
So that we can file tax returns and meet regulatory requirements.
Acceptance Criteria
Given controller selects report type (TDS, PF, ESI)
When they specify reporting period
Then system generates report in government-mandated format
Given report is generated
When exported
Then file format matches regulatory filing requirements (e.g., Form 24Q format)
Pet Care

Pet Day-Care Management Platform

PET-301
Feature High 8 pts
As a pet owner,
I want to book day-care slots for my pet online,
So that I can secure a spot without calling the facility.
Acceptance Criteria
Given pet owner is registered and logged in
When they navigate to booking calendar
Then available slots are shown with real-time capacity status
Given owner selects date and service type
When they confirm booking
Then slot is reserved and confirmation email sent with booking details
Given facility is at capacity
When owner tries to book
Then they see "Fully Booked" and option to join waitlist
PET-302
Feature Medium 5 pts
As a facility staff member,
I want to log daily activities and notes for each pet,
So that owners receive updates about their pet's day.
Acceptance Criteria
Given staff member has pet checked in
When they access pet's daily log
Then they can record: feeding times, play activities, bathroom breaks, mood/behavior
Given activity is logged
When staff adds photo
Then photo is attached to activity entry with timestamp
Given pet's day is complete
When owner is notified
Then they receive summary report card with all activities and photos
PET-303
Feature High 5 pts
As a facility manager,
I want to maintain vaccination records for all pets,
So that we ensure only vaccinated pets are admitted, protecting all animals.
Acceptance Criteria
Given new pet is being registered
When owner uploads vaccination documents
Then system stores documents and extracts vaccination dates
Given vaccination is nearing expiry (30 days)
When system runs daily check
Then automated reminder is sent to pet owner
Given pet has expired vaccinations
When owner tries to book
Then booking is blocked with message to update vaccination records
PET-304
Task Low 3 pts
As a pet owner,
I want to view live camera feed of the play area,
So that I can check on my pet during the day.
Acceptance Criteria
Given owner has active booking for the day
When they access "Live View"
Then they see live stream from designated camera(s)
Given owner has no active booking
When they try to access live view
Then access is denied with message "Available only during your pet's stay"
Marketplace

Vehicle Auction Platform

AUC-401
Epic High 13 pts
As a dealer,
I want to list my vehicle for auction with detailed specifications,
So that potential buyers have complete information to make bidding decisions.
Acceptance Criteria
Given dealer is verified and logged in
When they initiate new listing
Then they can enter: make, model, year, mileage, condition, VIN, and upload up to 20 photos
Given dealer sets auction parameters
When they configure listing
Then they specify: reserve price (optional), starting bid, auction duration, and buy-now price (optional)
Given listing is submitted
When approved by admin
Then vehicle appears in auction catalog at scheduled start time
AUC-402
Feature High 8 pts
As a buyer,
I want to place bids on vehicles in real-time,
So that I can compete for vehicles I'm interested in purchasing.
Acceptance Criteria
Given buyer is viewing active auction
When they enter bid amount above current bid + minimum increment
Then bid is recorded and displayed to all participants in real-time
Given buyer places a bid
When another bidder outbids them
Then they receive instant notification via app and email
Given bid is placed in final 2 minutes
When timer updates
Then auction extends by 2 minutes to prevent sniping
AUC-403
Feature Medium 5 pts
As a buyer,
I want to set up automatic bidding with a maximum limit,
So that the system bids on my behalf without exceeding my budget.
Acceptance Criteria
Given buyer wants to enable auto-bid
When they set maximum bid amount
Then system places minimum increment bids automatically when outbid
Given competing bid exceeds buyer's max limit
When auto-bid cannot match
Then buyer is notified their auto-bid limit was exceeded
Given two auto-bidders compete
When bids are processed
Then system resolves to winner's max at one increment above loser's max
AUC-404
Task Medium 5 pts
As a platform administrator,
I want to generate auction settlement reports,
So that financial transactions between buyers and sellers are properly documented.
Acceptance Criteria
Given auction has ended with winning bid
When admin generates settlement
Then report shows: final price, buyer fee, seller fee, platform commission
Given settlement report is finalized
When exported
Then invoices are automatically generated for buyer and seller
